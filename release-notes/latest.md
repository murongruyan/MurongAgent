更新摘要：本次版本重点打磨移动端交互体验、补充自定义斜杠命令系统和 BM25 智能检索，聊天与文件操作更流畅也更精确。

## 新增

- 新增自定义斜杠命令系统，项目下 `.murong/commands/` 目录中的 Markdown 文件自动注册为 `/` 命令，支持 `$1..$9` 和 `$ARGUMENTS` 参数展开。
- 新增 BM25 Okapi 全文检索，全局记忆和会话历史搜索从关键词包含升级为 BM25 排序，CJK 逐字分词，中文搜索更精准。
- 新增计划模式持久化，会话间保持计划开关状态，不再每次刷新重置。
- 新增 KeepShell 输出上限保护（1MB / 50K 行），防止大命令输出导致 OOM 闪退。
- 新增图片上传自动下采样（最长边 2048px，JPEG 85%），减少 token 浪费和内存压力。

## 优化

- 计划确认、澄清问题、Ask 用户三种弹窗从模态 Dialog 改为聊天内联卡片，手机端可自由滚动查看上下文。
- 发送前自动分流默认关闭，模型发现问题后直接写代码，不再先走规划判断。
- 通知服务刷新频率优化：面板不可见时降为 10s，协议轮询通知模式下 2s 低频运行。
- mode.txt 写入前缓存对比，减少 inotify 风暴和不必要的磁盘 I/O。
- 原生二进制状态缓存，避免每 500ms 跑 shell pidof 检查。
- 共享采样间隔从 500ms 降至 2s（仅通知模式），慢指标缓存从 1s 提到 3s。

## 修复

- 修复模型输出时滚动卡死：移除 LaunchedEffect 中每 token 变化的 key，防止 scrollToItem 被频繁取消。
- 修复全局记忆输入框光标跳位：移除 onValueChange 中的 .filter() 操作。
- 修复大文件读取闪退：readFile 增加 100KB 大小保护，超限提示改用分页读。
- 修复 ConfigRepository 启动 ANR：移除 init { runBlocking }，改为异步延迟迁移。
- 修复进程被杀后帧率悬浮球/悬浮胶囊/线程监视器不自动恢复。
- 修复 CI 编译失败：提交此前遗漏的 ProjectGuideFiles.kt 等 14 个文件。

## 稳定性

- 风驰避让时 FAS 预测线程完全暂停，GPU/DDR 强制立即返回，减少后台鬼火漏电。
- 退出游戏时日志不再误报"未检测到风驰"，改为"游戏退出"。
- 通知刷新只在用户查看面板时高频更新，离开面板暂停到 10s。
- 相关核心链路补充了回归测试覆盖。
