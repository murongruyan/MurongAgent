# MurongAgent 1.31

## 1.31 / 26072301 — 2026-07-23

本节只记录最新正式 Release `murong-suite-v1.30` 之后的变化。

### Added
- 新增统一设备连接体系。手机和电脑均拥有稳定的 16 位本机 ID，可通过同一 GitHub 账号、同一局域网自动发现、已授权 ADB、临时验证码、安全密码或接收端确认建立信任；已经配对的设备可直接重连，不再重复输入验证码。
- 新增双端连接申请与设备治理。手机通知和电脑系统通知都能同意、拒绝或拉黑申请，并提供已配对设备撤销、黑名单、免打扰、申请限速及过期清理。
- 新增公网稳定 ID Relay v2。设备先完成签名登记，再通过加密邀请与一次性隧道连接；普通用户界面不再要求填写 WSS、HTTP 地址或复制旧电脑连接码。
- 桌面端升级为完整 Agent 工作台：支持项目选择与新建、多标签持久终端、代码编辑与语法高亮、图片和 Markdown 预览、浏览器、Git、子代理、侧边聊天、项目文件树，以及可调整的侧栏、底部终端和聊天边距。
- 新增端到端聊天同步。手机与电脑可分页同步任意总量的会话历史；重复同步保持幂等，双方同时修改时保留冲突副本，不会静默覆盖原聊天记录。

### Changed
- 手机和电脑的远程控制页统一为“本机 ID、临时验证码、安全密码、已配对设备、输入设备码、当前环境设备”的直接操作结构；附近或已信任设备可点击“立即连接”。
- 电脑端关闭窗口时默认收进系统托盘，重复启动只激活已有窗口；正式图标同步用于主窗口、任务栏、托盘和系统预览。
- 桌面端设置改为分类居中的小弹窗，模型、规则、记忆、Skills、MCP 等按类型分组；任务详情和删除等会话操作移入右键菜单，摘要、侧栏和底部终端使用独立入口。
- Agent 官方服务部署收敛到 Go 后端和原子安装脚本，补齐令牌摘要、健康检查、失败回滚与生产环境审计，移除旧 PHP 迁移步骤和独立 Cloud Relay 服务。
- 完整构建工作流现在产出 Android 主程序、终端扩展，以及 Windows、macOS、Linux 的 amd64/arm64 桌面包，共 8 个正式包；废弃的两个独立 Cloud Relay 包不再发布。

### Fixed
- 修复大聊天记录同步时因单次 HTTP 读取边界而出现的 `unexpected EOF`。历史记录改为透明分页、逐页独立加密和校验，不再受单个固定总大小限制。
- 修复手机同步到电脑的 ChatGPT/Codex 账号被误判为普通 API 模型、空 Base URL 被当成非法地址的问题；官方账号会在电脑端使用内置默认端点。
- 修复 Codex `thread/start.dynamicTools` 缺少 `experimentalApi` capability、导致同步账号无法开始任务的问题。
- 修复异步切换模型时读取已销毁 DOM 节点造成的 `Cannot read properties of null (reading 'selectedOptions')`。
- 桌面聊天输入框补齐最近 50 条输入历史并持久化；第一次按上键会读取当前任务最近一次用户输入，同时保留尚未发送的草稿。
- 修复手机查看电脑聊天记录时使用额外弹窗、电脑新增消息未及时进入当前会话，以及同步后项目仍显示“未绑定项目”等状态不一致问题。
- 修复桌面编辑器左右布局、文件树字号、图片显示、Markdown 预览和代码高亮不完整的问题；终端直接打开所选本机 Shell，不再套用多余的特殊启动流程。
- 修复撤销配对只清理单端状态的问题。电脑会先通知手机撤销信任，再删除本机访问令牌和同步密钥；远端不可达时会保留本机凭据并明确提示失败。

### Security
- 设备身份使用 P-256 签名；首次连接使用临时 P-256 ECDH、HKDF-SHA256 与 AES-256-GCM 派生链路密钥。Relay 只转发有界密文，无法获得访问令牌或同步密钥。
- 临时验证码和安全密码使用带双 nonce 的 SCRAM-SHA-256。安全密码只保存 salt、迭代次数、StoredKey 与 ServerKey；临时验证码成功或连续失败后都会轮换并执行冷却。
- API Key、ChatGPT/Codex 登录令牌和 GitHub Token 只会在已配对设备之间的端到端加密同步包中传输；Android 使用 Keystore，Desktop 使用 DPAPI、Keychain 或系统凭据保护本地秘密。
- 设备配对只建立连接信任，不会绕过 Root、文件写入、终端、Git 或 GitHub 修改等原有工具审批。

## 1.30 / 26071902 — 2026-07-20

### Added
- 新增跨平台 Murong Desktop Agent：原生支持 Windows x64/ARM64、macOS Intel/Apple Silicon、Linux x64/ARM64，可选择或新建电脑本地项目，直接读写文件、编辑代码、查看 Diff，并调用 PowerShell、WSL、Bash 等本机终端。

- 电脑端补齐模型与推理强度、Skills、MCP、子代理、@ 文件/文件夹、目标模式和规范计划模式。规范计划必须用真实工具收据逐步签收，未完成步骤不能伪装成整体完成。
- 新增带版本清单、大小和 SHA-256 校验的完整备份与恢复。Android 与 Desktop 可安全合并可移植会话、普通配置、规则、记忆、Skills、MCP 和工作流；API Key、登录令牌、设备路径、终端扩展、语音模型和缓存默认排除。
- 新增项目外部变化追踪与跨任务审计。终端、Root Shell、Git、格式化器或其他编辑器造成的文件变化会在下一轮提供摘要，并可按项目追溯来源、路径、会话和结果。

### Changed
- 设置页将完整备份、远程 Web/电脑节点、GitHub 联动、运行后端和 MCP 等低频区域折叠整理，MCP 移到 Skills 下方，并移除重复的“关于”入口，减少手机端视觉噪声。
- 桌面与手机使用同一套会话、记忆、Skills、MCP、工作流和普通设置语义；用户明确选择同步凭据时，可通过第一方设备间的端到端加密通道同步 API Key 与登录令牌，普通备份仍不包含凭据。
- 构建链合并为一个“构建全部正式产物”入口，同时构建主 APK、终端扩展 APK、六个桌面包和两个云中继包；最终门禁要求 10 个包与逐包 SHA-256 全部准确无误。

### Fixed
- 语音输入恢复实时出字与自动标点，去除逐字之间的异常空格；长按期间轻微移动不再误停，上滑明确显示取消，手指未松开并滑回时继续录音，同时屏蔽录音期间的右滑切页。
- 终端扩展环境、Root Shell 与系统 Shell 的边界重新梳理；扩展 Bash 可在取得 Root 后继续使用完整工具链，不再因错误的 Shell 切换丢失环境。终端历史会持久化到磁盘，不再每次只剩 `exit`。
- 跨端新消息、规范计划进度和审批状态会同步刷新，手机无需退出会话才能看到电脑端新增记录。
- 朗读在页面滚动或普通点击后不再意外停止；只有用户主动暂停或播放结束才会终止。

## 1.10 / 26071620 — 2026-07-16

### Fixed
- **扩展环境仍被判定不可用**: 绝对目标 symlink 在运行时被跳过后，不再进入必需文件集合，避免环境完整性检查把无法创建且无需创建的链接当作缺失文件。
- **兼容扩展包 1.4**: 旧扩展 manifest 即使仍包含 Termux 绝对 keyring 链接，也可以正常安装并启用其余工具链。
- **POSIX `[` 命令被误判**: 工具链命令白名单明确接受标准 `[`/`test` 入口，不再因此拒绝整个扩展 manifest。
- **脚本命令被替换为自指断链**: 当命令入口和已解包脚本是同一路径时保留原文件，仅为 native 命令创建 `bin/` symlink。
- **`pkg`/`apt` 只有入口却无法安装**: 扩展环境通过 PRoot 映射官方 Termux 编译期前缀，并继承正确的包管理器环境；`pkg install`、`apt`、`dpkg` 不再被诊断脚本禁用。
- **高 targetSdk 下 PRoot 终端循环退出**: 首个 Termux ELF 通过 Android system linker 加载，`PROOT_LOADER` 使用扩展 APK 内原生库，避免应用数据目录 W^X 限制导致 `Permission denied` 和 `code 1` 重建。
- **动态安装命令仍被 W^X 拒绝**: 启用 `termux-exec` linker 模式，使 Python、pip、clang 等由 `pkg` 安装到工具链目录后的命令可以直接执行。
- **APT 仓库公钥链接缺失**: 扩展提供的绝对 keyring 目标正规化为工具链内相对链接，恢复仓库签名校验。
- **包管理器覆盖的命令被还原**: 工具链完整性修复只重建缺失或断开的入口，保留 `apt/dpkg` 已安装或升级的有效文件。
- **升级主程序后 APT 后装包消失**: APT 可删除的 `.murong-keep` 目录占位文件不再参与工具链完整性判定，避免冷启动误删并重建整个工具链，保留 Python、pip、clang 等后装包。
- **系统终端放大后泄漏颜色码**: 系统环境使用 Android `sh` 时改用无 ANSI 的提示符，终端尺寸变化不再显示 `39;172m` 等残缺控制序列；系统 Bash 仍保留彩色提示符。

### Changed
- **模型命令工具支持双环境**: `shell` 新增 `system` 与 `extension` 选择；系统环境继续使用 Root `su`，扩展环境以应用 UID 进入 PRoot/Termux，`pkg`/`apt` 安装的文件可与页面终端共同维护。
- **模型命令工作目录稳定化**: 主代理的 shell 命令默认固定到当前项目作用域，不再受两个持久 Root 会话随机复用及历史 `cd` 状态影响，也可通过 `working_directory` 显式覆盖。
- **工具链链接处理一致化**: manifest 验证、安装、缓存校验和最终可用性判断统一只处理可在应用沙箱内解析的相对目标链接。
- **工具链安装指纹升级**: 安装语义更新为 `relocatable-v2`，现有缓存会自动重建并应用包数据库、keyring 与 PRoot 兼容层。
- **项目运行器接入扩展包环境**: 自动安装依赖及 Python/Node 等项目命令与交互终端使用同一前缀映射，动态安装的命令可直接运行。
- **扩展运行时要求升级**: 完整包管理器环境要求扩展 manifest 提供 APK 内 `proot-loader` 入口；不满足时不会误报为可用包管理器。
